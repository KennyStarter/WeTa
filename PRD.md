# 产品需求文档（PRD）

## 朋友圈人物分析 H5 网页

**文档版本**：v1.0  
**创建日期**：2025-02-13  
**产品定位**：基于对方朋友圈截图，通过 AI 分析帮助用户与目标人物发展恋爱关系

---

## 一、需求核对与确认

### 1.1 核心需求理解

| 需求项 | 描述 | 状态 |
|--------|------|------|
| 产品形态 | H5 移动端网页 | ✓ 已确认 |
| 核心场景 | 用户想与某人发展男女朋友关系 | ✓ 已确认 |
| 输入内容 | 对方朋友圈截图，**5 张及以上** | ✓ 已确认 |
| 分析引擎 | 豆包大模型 doubao-seed-1.6-thinking | ✓ 已确认 |
| 输出目标 | 人物画像分析 + 交友/恋爱攻略建议 | ✓ 已确认 |

### 1.2 目标用户

- **主要用户**：对某人有好感，希望与其发展恋爱关系，但缺乏了解的人
- **使用场景**：在有一定朋友圈截图的前提下，希望快速了解对方性格、喜好、生活方式，并得到可操作的交友/追求建议

### 1.3 产品目标（更明确的定位）

> **帮助用户通过分析对方朋友圈，形成「人物画像 + 追求攻略」，从而更有效、更得体地与目标对象发展男女朋友关系。**

---

## 二、功能需求

### 2.1 功能模块总览

**单页设计**：所有功能集中在一个上传页面内完成，上传、加载、结果展示均在同一页。

```
┌─────────────────────────────────────────────────────────────┐
│              朋友圈人物分析 H5（单页·上传页）                  │
├─────────────────────────────────────────────────────────────┤
│  1. 图片上传   │  2. 分析请求    │  3. 结果展示               │
│  - 上传 5+ 张  │  - 调用豆包 API │  - 人物画像 + 追求攻略     │
│  - 预览/删除   │  - 深度思考模式 │  - 同页内展示，支持滚动    │
│  - 格式校验    │  - 同页加载态   │                            │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 详细功能说明

#### 模块一：图片上传

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 上传入口 | 支持点击上传、拖拽上传 | P0 |
| 数量限制 | 最少 5 张，建议 5–20 张 | P0 |
| 格式限制 | 支持 JPG、PNG、WebP | P0 |
| 单张大小 | 建议 ≤ 5MB（根据豆包 API 限制） | P0 |
| 预览与删除 | 上传后可预览，支持删除单张 | P0 |
| 顺序调整 | 可选：支持调整图片顺序 | P2 |

#### 模块二：分析请求

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 提交分析 | 点击「开始分析」触发 | P0 |
| 调用豆包 API | 使用 doubao-seed-1.6-thinking，多模态图片理解 | P0 |
| 深度思考 | 启用深度思考模式，提高分析质量 | P0 |
| 加载态 | 分析中显示进度/加载动画（预估 30s–2min） | P0 |
| 错误处理 | API 失败、超时、鉴权失败等提示 | P0 |

#### 模块三：结果展示

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 人物画像 | 性格、爱好、生活状态、价值观等 | P0 |
| 追求攻略 | 聊天话术建议、约会建议、注意事项 | P0 |
| 结构化展示 | 分段、分块展示，便于阅读 | P0 |
| 分享/保存 | 可生成图片或链接分享（可选） | P2 |

### 2.3 分析 Prompt 设计（核心逻辑）

#### System Prompt（角色设定）

调用豆包 API 时使用的 system 消息：

```
你是一个追求异性的高手，对于青春男女生的心理和外在表现，有非常强的洞察，也有一套很厉害的追求异性的技巧！擅长于输出简短但有效的分析和建议。
```

#### User Prompt（分析指令示意）

与图片一起提交的用户消息（可结合实际实现调整）：

```
用户上传了多张某人的朋友圈截图，希望了解此人并与之发展恋爱关系。
请基于这些朋友圈截图，输出简短有效的人物画像和追求攻略建议。
```

## 三、技术方案

### 3.1 豆包 API 接入

#### 使用模型

- **模型 ID**：`doubao-seed-1.6-thinking`
- **能力**：多模态理解（图文）、深度思考
- **文档**：https://www.volcengine.com/docs/82379/1399008

#### 调用方式

- **接口**：ChatCompletions - 视觉理解 API
- **文档**：https://www.volcengine.com/docs/82379/1362913  
- **Responses API 多模态**：https://www.volcengine.com/docs/82379/1958521

#### 鉴权与配置

- 需在火山方舟控制台创建 **API Key**
- 使用 API Key 进行鉴权，**不能在前端暴露**
- 需部署 **后端代理服务** 转发请求

### 3.2 架构建议

```
┌──────────────┐      ┌──────────────┐      ┌──────────────────┐
│   H5 前端     │ ──→  │  后端代理     │ ──→  │  火山方舟 API    │
│  - 上传图片   │      │  - 鉴权       │      │  doubao-seed-    │
│  - 展示结果   │ ←──  │  - 转发请求   │ ←──  │  1.6-thinking    │
└──────────────┘      └──────────────┘      └──────────────────┘
```

**原因**：API Key 必须由服务端保管，前端只能调用自有后端。

### 3.3 图片处理

- 前端：压缩、格式校验、Base64 或 FormData 上传
- 豆包视觉 API：支持 Base64 或图片 URL 输入
- 多图：在 messages 中按顺序附带多张图片 + 统一分析指令

### 3.4 非功能需求

| 项目 | 说明 |
|------|------|
| 响应时间 | 深度思考模式下约 30s–2min，需合理超时设置 |
| 兼容性 | 支持 iOS Safari、Android Chrome、微信内置浏览器 |
| 隐私 | 明确提示：截图仅用于分析，不存储、不分享 |
| 安全 | API Key 仅存在于后端，前端无敏感信息 |

## 四、界面与交互

### 4.1 页面流程

**单页流程**：全程在同一页面内完成，无页面跳转。

```
上传页 → 上传 5+ 张 → 点击「开始分析」 → 同页加载态 → 同页展示结果
```

### 4.2 关键页面元素（单页布局）

- **上传区**：上传入口、数量提示（至少 5 张）、已选图片预览、开始分析按钮
- **加载态**：分析中时，同页展示进度条或加载动画，预估时长提示
- **结果区**：分析完成后，同页下方展示人物画像、追求攻略，支持滚动阅读

## 五、风险与合规

### 5.1 风险点

| 风险 | 说明 | 应对 |
|------|------|------|
| 隐私与合规 | 未经同意分析他人朋友圈可能涉及隐私 | 在页内显著位置做合规与隐私说明 |
| API 成本 | 多模态 + 深度思考调用成本较高 | 做好用量与费用监控 |
| 结果准确性 | AI 分析存在偏差 | 在结果中加免责声明，提示仅供参考 |

### 5.2 合规建议

- 在显著位置说明「本工具仅供个人了解参考，不构成专业建议」
- 明确「请勿在未经他人同意的情况下过度收集其隐私信息」
- 结果页增加免责声明
